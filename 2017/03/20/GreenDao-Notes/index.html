添加数据

添加数据很简单，只要创建一个对象，并调用UserDao 的insert 方法即可。
创建对象时传进去的两个参数即实体类的两个参数，null 将使id默认为递增。

删除数据

删除数据也不难，通过调用queryBuilder() 得到QueryBuilder 对象，通过它查询要删除的对象，其中unique() 表示得到一个单一对象，若为list() 则表示得到符合条件的对象的集合，当不希望返回null作为结果时，则调用uniqueOrThrow()方法，当结果null时将直接抛出异常。最后再调用delete() 方法即可。

查询数据

查询数据其实在上面删除和更新数据已经用过了，但还有一些更高级的方法没展开，下面将主要对query方法进行介绍。

GreenDao查询方法详解

order,limit,offset

orderAsc 是按升序排列，相反的，Desc 则是按降序排列。此外还可以自定义排序orderCustom。
limit 是将查询到的数据限制为几条。
offset 需配合limit 使用，它是经limit后查询到的数据，跳过前面几条后得到最后的查询结果。举个例子，查询到的数据为“1，2，3，4，5”，若offset(2)，则最后查询到数据为“3，4，5”。

多线程查询

从greenDAO 1.3版本开始，实例化Query对象均会绑定到其自己的线程，这样我们就可以安全的设置Query参数，因为其他线程无法影响到。如果其他线程使用别的线程的query（比如设置参数、执行查询）将会抛出异常。因此，我们也不需要做同步工作，而且不要加锁，因为加入相关的事务用到了该Query对象将导致死锁。为完全地避免可能产生的死锁，1.3版本引入了forCurrentThread()方法。该方法将返回本线程内的Query实例，每次调用该方法时，参数均会被重置为最初创建时的一样。

LazyList

list(): 所有查询结果均会被加载到内存中，返回一个ArrayList。
listLazy(): 查询结果会根据需要加载到内存中。列表中的元素仅在accessed for the first time，它才会被加载并缓存。该方法必须主动关闭（Must be closed）。
